# 다시나 (Dasina) — 말벗 앱 개발자 가이드

## 전체 풀 과정 상세 문서

---

## 1. 앱 개요

**다시나**는 뇌졸중 후 마비말장애(dysarthria) 환자를 위한 재활 앱이다.
두 개의 하위 앱으로 구성된다:

- **말벗**: 언어 재활 (마비말장애 치료)
- **마음벗**: 우울증 치료 (정서 연동)

### 핵심 원칙
1. **비판 금지** — 실패해도 부정적 피드백 없음
2. **속도보다 반복** — 매일 하는 것이 핵심
3. **감정 케어 필수** — 우울도에 따라 훈련 강도 자동 조절

---

## 2. 사용자 플로우 (Full Flow)

```
[초기 설정] → [홈] → [rPPG 측정] → [훈련 목록] → [운동 실행] → [세션 완료]
     ↓            ↓         ↓              ↓             ↓            ↓
   성함       모드 선택   우울도 판정    과제 조합 결정   Phase A→B/C   결과+레벨변동
   생년       통합/집중   HRV 분석      레벨별 목표     자동측정/자가    스탬프+공유
   발병일                               표시           확인
```

### 2.1 초기 설정 (Screen: setup)

첫 실행 시 1회 표시. 3가지 정보 수집:

| 필드 | 타입 | 용도 |
|------|------|------|
| 성함 | text input | 홈 화면 인사말 |
| 태어난 해 | select (드롭다운) | 나이 계산 |
| 뇌졸중 진단일 | date input | 병기 판정 + D-day 계산 |

**병기 판정 로직:**

```javascript
경과일 = (오늘 - 발병일) / 86400000  // ms → days

if (경과일 <= 30)  → 급성기     → 진단 2주마다, 훈련 매일 30~40분
if (경과일 <= 90)  → 아급성기   → 진단 4주마다, 훈련 매일 40~50분
if (경과일 <= 180) → 만성기     → 진단 8주마다, 훈련 주5회 30~40분
if (경과일 > 180)  → 유지기     → 진단 12주마다, 훈련 주3회 이상
```

**골든타임 D-day:**
- 180일 기준 카운트다운
- 홈 화면 상단에 배지 표시
- 급성기=빨강, 아급성기=주황, 만성기=회색

### 2.2 홈 화면 (Screen: home)

**모드 선택:**

| 모드 | 변수값 | 설명 |
|------|--------|------|
| 통합 재활 | `mode='integ'` | rPPG 우울도 + 중증도 연동 |
| 집중 재활 | `mode='focus'` | 우울 정상 전제, 중증도만으로 풀 세트 |

**표시 요소:**
- 환자명 인사말
- D-day 골든타임 배지
- 연속 훈련일 (streak)
- 모드 배지
- 메뉴 카드 3개 (진단, 훈련, 기록)

### 2.3 rPPG 측정 (Screen: rppg)

카메라 기반 비접촉 심박 측정 → 심박변동성(HRV) → 우울/스트레스 수준 판정.

**측정 흐름:**
1. 안내 화면 → "측정 시작" 클릭
2. 30초 카운트다운 + 실시간 BPM/HRV 표시
3. 결과 화면: 우울 수준 + 훈련 처방 미리보기

**결과 매핑:**

| HRV | 심박 | 우울 수준 | 변수 | 훈련 강도 |
|-----|------|----------|------|----------|
| 18ms | 88bpm | 높음 | `dep='high'` | 최소 과제 |
| 38ms | 74bpm | 보통 | `dep='mid'` | 균형 훈련 |
| 58ms | 65bpm | 낮음 | `dep='low'` | 최대 도전 |

**집중 재활 모드 시:**
- rPPG는 실행하지만 결과는 항상 "정상 — 스트레스 없음" 표시
- HRV 62ms 고정

### 2.4 훈련 목록 (Screen: tasks)

rPPG 결과(dep) + 중증도(sev) 조합으로 오늘의 훈련 세트 결정.

**조합 키:** `COMBOS[dep + '-' + sev]`

#### 통합 모드 COMBOS (9가지 조합)

| 조합 | 과제 수 | 시간 | 전략 |
|------|---------|------|------|
| high-severe | 3 | 5~7분 | 최소 과제 + 기초 유지 |
| high-moderate | 4 | 7~10분 | 기초 발성 + 성공 경험 |
| high-mild | 4 | 8~10분 | 가벼운 발성 |
| mid-severe | 5 | 10~13분 | 기초 + 발성 |
| mid-moderate | 5 | 12~15분 | 발성 + 조음 균형 |
| mid-mild | 6 | 13~15분 | 전 영역 골고루 |
| low-severe | 6 | 15~18분 | 강도 있는 기초 |
| low-moderate | 7 | 18~20분 | 전 영역 도전 |
| low-mild | 9 | 20~25분 | 최대 세트 |

#### 집중 모드 FOCUS_COMBOS (3가지)

| 조합 | 과제 수 | 시간 |
|------|---------|------|
| normal-severe | 16 (전체) | 40~50분 |
| normal-moderate | 12 | 30~35분 |
| normal-mild | 9 | 25~30분 |

**각 COMBO 데이터 구조:**

```javascript

  dur: "5~7분",                     // 예상 시간
  ids: [1, 12, 2],                  // 운동 ID 배열
```

### 2.5 운동 실행 (Screen: run)

각 운동은 다음 흐름으로 실행된다:

```
[Phase A] 영상 시청 → [Phase B/B-read/C] 과제 수행 → "다음으로" → 레벨 체크 → 다음 운동
```

#### Phase 분기

| 운동 타입 | Phase | 설명 |
|----------|-------|------|
| `t:'self'` (자가확인) | Phase C | 체크리스트 확인 |
| `t:'auto'` + READ_EX에 있음 | Phase B-read | 읽기 과제 (5종) |
| `t:'auto'` + 그 외 | Phase B | 음성 녹음 + 자동 측정 |

## 3. 레벨 시스템 (Level System)

### 3.1 핵심 원칙

**80% 규칙 (Motor Learning Theory, Maas et al. 2008):**
- 3회 연속 목표 달성 → 자동 레벨업
- 3회 연속 목표 실패 → 자동 레벨다운
- 성공↔실패 전환 시 연속 카운트 리셋

### 3.2 데이터 구조

### 3.4 레벨 체크 로직 (구현 코드)

```javascript
function checkLevelChange(exerciseId) {
  if (!LEVELS[exerciseId]) return;

  // 실제 앱: 측정값 vs 레벨 목표 비교
  // const measured = getLastMeasurement(exerciseId);
  // const goal = getLvl(exerciseId);
  // const success = measured >= goal.max;

  const success = /* 측정값이 목표 달성 여부 */;

  // 연속 카운트 업데이트
  if (success) {
    lvStreak[id] = lvStreak[id] >= 0 ? lvStreak[id] + 1 : 1;
  } else {
    lvStreak[id] = lvStreak[id] <= 0 ? lvStreak[id] - 1 : -1;
  }

  // 3회 연속 성공 → 레벨업
  if (lvStreak[id] >= 3 && playerLvl[id] < maxLevel) {
    playerLvl[id]++;
    lvStreak[id] = 0;
    showLevelUpNotification();
  }

  // 3회 연속 실패 → 레벨다운
  if (lvStreak[id] <= -3 && playerLvl[id] > 1) {
    playerLvl[id]--;
    lvStreak[id] = 0;
    showLevelDownNotification(); // "괜찮아요, 천천히 다시 쌓아가요"
  }
}
```

### 3.5 레벨 시스템 × 기분 조정 관계

```
┌─────────────────────────────────────────────────────┐
│                 2축 독립 조절 시스템                    │
│                                                     │
│  [기분 축 — rPPG]          [난이도 축 — LEVELS]       │
│  우울 높음 → 과제 3개       각 운동마다 Lv.1~5        │
│  우울 보통 → 과제 5~6개     3회 연속 달성 → 자동 승급   │
│  우울 낮음 → 과제 7~9개     3회 연속 실패 → 자동 강등   │
│                                                     │
│  → "몇 가지" 할지 결정      → "얼마나 어렵게" 할지 결정  │
└─────────────────────────────────────────────────────┘
```

**예시: 우울 높은 날 + Lv.3 환자**
- 3가지 운동만 수행 (과제 수 줄임)
- 하지만 각 운동의 목표는 Lv.3 유지 (난이도는 환자 실력에 맞춤)
- 만약 우울로 인해 3회 연속 실패 → Lv.2로 자동 강등

## 4. 읽기 과제 시스템 (Phase B-read)

읽기 과제는 5가지 서브타입으로 구분된다:

| 서브타입 | 운동 ID | 화면 구성 | 측정 방식 |
|---------|--------|----------|----------|
| slow (슬로우) | #14 | 문장 카드 + 음절 하이라이트 | 말속도 (어/분) |
| loud (라우드) | #15 | 문장 카드 + dB 미터 | 음량 (dB) |
| pair (대조) | #10 | 최소대립쌍 카드 2개 | STT 정확도 |
| tap (탭핑) | #13 | 비트 박자 + 문장 | 타이밍 정확도 |
| stress (대립강세) | #11 | 강조 단어 하이라이트 문장 | 강세 패턴 정확도 |

**매핑:**
```javascript
const READ_EX = { 10:'pair', 13:'tap', 14:'slow', 15:'loud', 11:'stress' };
```

## 6. 평가 체계

### 6.1 초기 평가 (중증도 판정)

3가지 과제로 4개 영역 측정:

| 평가 과제 | 측정 영역 | 측정 항목 |
|----------|----------|----------|
| 아, 이, 우 모음발성 | 호흡 + 발성 | MPT, 음질(HNR/jitter/shimmer), F0, 음량 |
| 퍼터커 DDK | 조음 | 음절 반복 속도, 정확도 |
| 단어읽기 30개 | 조음 + 운율 | 명료도, 조음 정확도, 말속도 |

**중증도 분류:**
- 중도 (severe)
- 중등도 (moderate)
- 경도 (mild)

### 6.2 미니 평가 (매 세션)

세션 시작 전 60초 미니 평가:
- "아~~~" 3초
- /퍼터커/ 5회
- 기준 단어 5개

→ 일별 추이 비교용 데이터 축적

### 6.3 재평가 주기

병기에 따라 자동 결정 (초기 설정의 발병일 기준):

| 병기 | 재평가 주기 | 미니 평가 |
|------|-----------|----------|
| 급성기 (0~30일) | 2주마다 | 매 세션 |
| 아급성기 (31~90일) | 4주마다 | 매 세션 |
| 만성기 (91~180일) | 8주마다 | 매 세션 |
| 유지기 (180일+) | 12주마다 | 매 세션 |

## 10. 프로토타입 테스트 가이드

### 프로토타입 컨트롤 바 (상단 검정 바)

| 컨트롤 | 버튼 | 설명 |
|--------|------|------|
| 우울(rPPG) | 높음/보통/낮음 | dep 변수 변경 → 과제 수/시간 변경 |
| 중증도 | 중도/중등도/경도 | sev 변수 변경 |
| 화면 | 설정/홈/rPPG/훈련목록/운동실행/결과 | 화면 직접 이동 |
| 모드 | 통합/집중 | 재활 모드 전환 |
| 레벨 | Lv.1~4 | 전체 운동 레벨 일괄 변경 |

### 테스트 시나리오

**시나리오 1: 우울 높은 중증 환자**
1. 우울: 높음, 중증도: 중도 선택
2. 훈련목록 → 3가지 과제만 표시 확인
3. 세션 시작 → 각 운동 "다음으로" 클릭
4. 결과 화면 확인

**시나리오 2: 레벨업 확인**
1. 레벨: Lv.1 설정
2. 세션 여러 번 반복 (각 운동에서 "다음으로" 클릭)
3. 70% 확률로 성공 → 3회 연속 시 레벨업 토스트 확인
4. 결과 화면에서 "레벨 변동" 섹션 확인

**시나리오 3: 집중 재활 모드**
1. 모드: 집중 선택
2. 중증도: 중도 → 16가지 전체 표시 확인
3. rPPG 결과 → "정상" 고정 확인

**시나리오 4: 초기 설정**
1. 페이지 새로고침
2. 설정 화면에서 이름/생년/발병일 입력
3. 발병일에 따른 병기 판정 카드 확인
4. "시작하기" → 홈 화면 이동 + D-day 업데이트 확인
